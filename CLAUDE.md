## Repository Overview

This is a NixOS desktop environment configuration repository that manages system-level configuration, application installations, and dotfiles for a Hyprland-based desktop setup with Japanese locale settings.

## NixOS Configuration Architecture

### Core Configuration Structure

The NixOS configuration is modular and split across several files:

- `nixos/configuration.nix` - Main system configuration (bootloader, networking, desktop environment, virtualization)
- `nixos/app-configuration.nix` - All package installations organized by category (GUI tools, terminal utilities, development tools)
- `nixos/user-configuration.nix` - User-specific settings (created locally, not in repo)
- `nixos/env-configuration.nix` - Environment-specific settings (firewall ports, Dropbox service)
- `/etc/nixos/hardware-configuration.nix` - Hardware-specific config (generated by NixOS, not in repo)

The main configuration imports these modules conditionally using `lib.optional` for user-configuration and direct imports for others.

### Configuration Linking

The repository expects to be cloned to `~/myconfig/config/` and symlinked to system locations:
- `sudo ln -s ~/myconfig/config/nixos/configuration.nix /etc/nixos/configuration.nix`
- `sudo ln -s ~/myconfig/config/nixos/app-configuration.nix /etc/nixos/app-configuration.nix`

User-specific files (`user-configuration.nix`, `env-configuration.nix`) must be created in `/etc/nixos/` manually.

## Common Commands

### NixOS System Management

```bash
# Apply configuration changes
sudo nixos-rebuild switch

# Garbage collection (remove generations older than 30 days)
sudo nix-collect-garbage --delete-older-than 30d

# Regular garbage collection
sudo nix-collect-garbage
```

### NixOS Channel Information

The system uses nixos-24.05 channel:
```bash
sudo nix-channel --list
# Expected: nixos https://nixos.org/channels/nixos-24.05
```

### Application Configuration Linking

Dotfiles are symlinked from the repository:
```bash
ln -s ~/myconfig/config/bash/.bashrc ~/.bashrc
ln -s ~/myconfig/config/.vimrc ~/.vimrc
ln -s ~/myconfig/config/alacritty ~/.config/alacritty
ln -s ~/myconfig/config/hypr ~/.config/hypr
ln -s ~/myconfig/config/eww ~/.config/eww
```

## System Configuration Details

### Desktop Environment

- Primary desktop: Hyprland (Wayland compositor)
- Display manager: SDDM with autologin
- Also includes: KDE Plasma 6
- Compositor configs split into modular files in `hypr/config/`:
  - `general.conf` - General Hyprland settings
  - `key.conf` - Keybindings
  - `monitors.conf` - Monitor configuration
  - `local-monitors*.conf` - Multiple local monitor configurations
  - `envars.conf` - Environment variables
  - `session.conf` - Session settings
  - `windowrules.conf` - Window rules

### Locale & Input

- Timezone: Asia/Tokyo
- Locale: ja_JP.UTF-8
- Input method: fcitx5 with Mozc
- Keyboard: Japanese (jp106) with Caps Lock mapped to Ctrl
- Dual boot friendly: Hardware clock set to local time

### Virtualization

- Docker enabled
- VirtualBox enabled with hardening disabled
- Custom udev rules and systemd slice configuration for VirtualBox
- User must be in `vboxusers` group

### Special Configurations

- Chrome password store set to "basic" via environment variable
- DisplayLink support enabled for external monitors
- Nix flakes and nix-command enabled
- Kernel: Linux 6.12 (linuxPackages_6_12)

## Directory Structure

## Important Notes for Modifications

### Adding Packages

Add new packages to `nixos/app-configuration.nix` in the appropriate category (GUI Tools, Terminal Utilities, System & System Utilities, or For Dev).

### Modifying System Settings

System-level settings go in `nixos/configuration.nix`. User-specific settings should go in `/etc/nixos/user-configuration.nix` (not tracked in repo).

### Hyprland Configuration

The main `hypr/hyprland.conf` sources modular config files. Edit the specific config file rather than the main file:
- Monitor changes → `hypr/config/monitors.conf` or `local-monitors*.conf`
- Keybindings → `hypr/config/key.conf`
- Window rules → `hypr/config/windowrules.conf`

### Testing Configuration Changes

Always test NixOS configuration changes with `sudo nixos-rebuild switch`. The system will validate the configuration before applying.
